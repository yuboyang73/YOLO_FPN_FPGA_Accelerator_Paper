## 4. 遇到的问题与处理

### Top Function 配置错误

曾出现：

```text
Cannot find function 'streaming_frontend_axis'
```

原因是 Solution 的 `directives.tcl` 仍然指定旧 Top Function `streaming_frontend`。已修正为：

```tcl
set_directive_top -name streaming_frontend_axis "streaming_frontend_axis"
```

修正后 C Simulation 和 C Synthesis 均可运行。

### Cosimulation 超时

重构后的设计虽然取消了整帧输入/输出 BRAM，但仍需 4 次重放输入帧，完整 RTL 仿真事件量较大。已将 Cosimulation 脚本的 trace level 改为 `none`，但本次仍未在约 5 分钟内完成。下一步应先增加小尺寸 smoke test，而不是直接反复运行完整配置。

## 5. 必会知识点

- Line Buffer 负责保存相邻行数据，使 3x3 窗口能够逐列滑动；
- Window Buffer 负责形成连续的 3x3 局部窗口；
- partial sum 用于在输出通道折叠时跨输入通道保存部分累加结果；
- AXI4-Stream 通过 `TVALID/TREADY` 完成数据传输握手；
- C Simulation 只能证明数值和功能；
- C Synthesis 才能查看 HLS 估计资源、延迟、时钟和 II；
- C/RTL Cosimulation 用于验证综合后 RTL 与 C 模型的时序级一致性；
- `II=1`、5 ns 时序闭合和 Cosimulation PASS 必须分别提供报告证据。
