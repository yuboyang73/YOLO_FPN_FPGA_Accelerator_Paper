## Interpretation

The previous implementation stored all 64 x 1444 x 3 x 3 valid windows,
which caused 1025 BRAM_18K blocks and exceeded the 280 BRAM_18K blocks
available on the Zynq-7020. The revised implementation constructs one 3x3
window on demand and immediately performs the MAC. This removes the full
window storage and reduces the estimated BRAM use to zero while preserving
the golden output exactly.

This is a memory-safe functional/HLS baseline. It is not yet the final
AXI4-Stream architecture: the current top-level interface remains
`ap_memory`, and the MAC accumulation loop still has II=18. Further work is
required to introduce a true line-buffer/window-buffer dataflow and reduce
the effective initiation interval toward II=1.
