# 阶段一实验全流程整理：YOLO Baseline、FPN 分支裁剪与 Golden 数据导出

> 本文档整理阶段一“YOLO Baseline 建立与信息熵裁剪”的完整实验链路，说明每一步的实验目的、实际作用、论文中的体现方式、背后的知识点以及当前已有证据文件。  
> 当前整理基于本地工作区截至 2026-07-10 的实验产物，未执行远端同步。

---

## 1. 阶段一的总体目标

阶段一的核心目标不是单纯训练一个 YOLO 模型，而是建立一条可以支撑后续 HLS 加速设计的算法端证据链：

```text
数据集准备
    ↓
YOLOv8n baseline 训练
    ↓
FPN/PAN 融合结构分析
    ↓
跳连分支信息量评估
    ↓
Entropy / Random / L1 三类裁剪对照实验
    ↓
Table 2 固化算法端消融结果
    ↓
导出 HLS 所需 golden_input / golden_output / weights
```

这条链路在论文中承担两个作用：

1. 在算法端说明：FPN 中存在可裁剪的跨层分支，裁剪后模型精度仍然可维持在 baseline 附近。
2. 在硬件端说明：后续 HLS 实现不是凭空选一个卷积，而是从裁剪后的关键 FPN 融合路径中抽取算子进行定点化和流式加速。

需要特别注意：当前实验结果显示，信息熵裁剪与随机裁剪均值非常接近，并没有形成“信息熵显著优于随机裁剪”的强证据。因此论文中应谨慎表述为“信息熵裁剪能够在当前设置下保持 baseline 附近的检测性能，并提供一种可解释的分支选择依据”，不要写成“明显优于随机剪枝”。

## 3. Step 1：数据集整理与 YOLO 格式准备

### 这一步做了什么

将 NEU-DET 缺陷检测数据整理为 YOLO 可训练格式，包括图像、标签、类别名称和 train/val/test 路径。

当前项目使用的类别为：

```text
crazing
inclusion
patches
pitted_surface
rolled-in_scale
scratches
```

### 这一步的作用

这一步决定后续所有实验是否公平。Baseline、Entropy、Random、L1 必须使用同一套数据划分、同一图像分辨率和同一评价指标，否则 Table 2 的比较没有意义。

### 论文中的体现

论文中对应 Chapter 4 §4.1 Experimental Setup，可写入：

- 数据集名称：NEU-DET。
- 类别数量：6 类。
- 输入分辨率：640 × 640。
- 评价指标：mAP@0.5 和 mAP@0.5:0.95。
- 训练/验证划分方式：如果最终论文中要写精确比例，需要从数据处理脚本或 dataset YAML 中再次确认。

可作为 Table 1 的内容：

| Class | Images / Instances | Split |
|---|---:|---|
| crazing | to be filled | train / val / test |
| inclusion | to be filled | train / val / test |
| patches | to be filled | train / val / test |
| pitted_surface | to be filled | train / val / test |
| rolled-in_scale | to be filled | train / val / test |
| scratches | to be filled | train / val / test |

### 背后的知识点

- YOLO 标签格式：`class_id x_center y_center width height`，坐标为归一化比例。
- 目标检测评价指标：Precision、Recall、mAP@0.5、mAP@0.5:0.95。
- 数据集划分必须固定，否则不同实验之间的 mAP 不能直接比较。

## 4. Step 2：YOLOv8n Baseline 训练与验证

### 这一步做了什么

使用 YOLOv8n 在 NEU-DET 数据集上训练 baseline 模型，并记录 baseline 的精度、参数量和计算量。

当前 Table 2 采用统一模型解析口径后的 baseline 结果：

| Model | mAP@0.5 | mAP@0.5:0.95 | Parameters | GFLOPs |
|---|---:|---:|---:|---:|
| Baseline YOLOv8n | 0.759 | 0.447 | 3,012,018 | 8.2 |

### 这一步的作用

Baseline 是阶段一所有裁剪实验的参照物。没有 baseline，就无法判断裁剪后模型是精度下降、保持还是提升。

### 论文中的体现

对应 Chapter 4 §4.2 Baseline and Pruning Ablation。

可以写成：

```text
The YOLOv8n baseline achieved 0.759 mAP@0.5 and 0.447 mAP@0.5:0.95 on the NEU-DET validation set. This model was used as the reference for all subsequent pruning experiments.
```

### 背后的知识点

- Baseline 是消融实验的零点。
- 参数量 Params 反映模型权重规模。
- GFLOPs 反映理论计算量，但 Ultralytics 输出通常保留一位小数，因此 8.2 到 8.1 的变化要谨慎解释。
- mAP@0.5 更宽松，mAP@0.5:0.95 更严格，更能反映定位质量。

## 5. Step 3：FPN/PAN 结构定位

### 这一步做了什么

分析 YOLOv8n 的 Neck 部分，定位所有 Concat 融合层及其输入来源。

当前识别到的 Concat 层包括：

| Concat Layer | From | 含义 |
|---:|---|---|
| 11 | `[-1, 6]` | 上采样主路径与 Layer 6 skip branch 融合 |
| 14 | `[-1, 4]` | 上采样主路径与 Layer 4 skip branch 融合 |
| 17 | `[-1, 12]` | 下采样主路径与 Layer 12 branch 融合 |
| 20 | `[-1, 9]` | 下采样主路径与 Layer 9 branch 融合 |

### 这一步的作用

信息熵裁剪必须先知道“哪些分支可以被裁剪”。YOLO 的 FPN/PAN 结构不是单链路，而是由多尺度上采样、下采样和 Concat 构成。如果不先定位 Concat 输入来源，就无法定义分支级别的裁剪对象。

### 论文中的体现

对应 Chapter 3 §3.2 Entropy-Guided FPN Branch Pruning。

论文中应把 FPN 裁剪对象说清楚：

```text
The pruning candidates are defined as the input branches of the Concat layers in the YOLOv8n neck. Each candidate branch corresponds to an intermediate feature stream participating in multi-scale feature fusion.
```

### 背后的知识点

- FPN：自顶向下融合高层语义和低层空间细节。
- PAN：进一步引入自底向上的路径增强。
- Concat：在通道维度拼接不同尺度或不同路径的特征。
- Skip connection：保留早期层特征，但会增加中间特征图缓存和跨层搬运压力。

## 6. Step 4：分支特征图 Hook 与信息熵计算

### 这一步做了什么

使用 forward hook 捕获各 Concat 输入分支的特征图，并对每个分支计算平均空间信息熵。

当前 `all_concat_branch_entropy.txt` 中的排序结果如下：

| Rank | Concat | Branch | Source Layer | Shape | Mean Entropy | Activation Elements |
|---:|---:|---:|---:|---|---:|---:|
| 1 | 20 | 0 | 19 | `(1,128,20,20)` | 5.580449 | 51,200 |
| 2 | 17 | 0 | 16 | `(1,64,40,40)` | 5.685125 | 102,400 |
| 3 | 11 | 1 | 6 | `(1,128,40,40)` | 5.792360 | 204,800 |
| 4 | 14 | 1 | 4 | `(1,64,80,80)` | 6.070816 | 409,600 |
| 5 | 14 | 0 | 13 | `(1,128,80,80)` | 6.265172 | 819,200 |
| 6 | 17 | 1 | 12 | `(1,128,40,40)` | 6.265172 | 204,800 |
| 7 | 11 | 0 | 10 | `(1,256,40,40)` | 6.412962 | 409,600 |
| 8 | 20 | 1 | 9 | `(1,256,20,20)` | 6.412962 | 102,400 |

其中最终用于信息熵裁剪的跳连分支为：

```text
Layer 6 -> Concat 11
```

### 这一步的作用

这一步把“裁剪哪个分支”从主观选择变成可计算的排序问题。信息熵用于衡量特征图空间分布的信息丰富程度。平均熵较低的分支在当前样本集上表现出较低的空间信息变化，可作为候选冗余分支。

需要注意：当前实际选择并不是简单裁掉全局 rank=1 的所有分支，而是在可解释的 FPN skip branch 候选中选择低熵分支。因为部分低熵分支属于主路径，如果直接裁掉可能破坏结构语义或造成不公平比较。

### 论文中的体现

对应 Chapter 3 §3.2 的公式、Algorithm 1 和 Figure 2。

可写入公式：

```text
For the k-th FPN branch, let X^(k) ∈ R^(C_k × H_k × W_k) denote its feature map. For each channel, a histogram-based probability distribution is estimated over spatial activations, and the channel entropy is computed as:

H_c^(k) = - Σ_b p_c(b) log2(p_c(b) + ε)

The branch-level entropy is then obtained by averaging over all channels:

H_bar^(k) = (1 / C_k) Σ_c H_c^(k)
```

### 背后的知识点

- Hook：在不改模型 forward 主体的情况下捕获中间层输入/输出。
- 信息熵：用于衡量分布不确定性或信息丰富程度。
- 直方图估计：把连续激活值离散成 bin 后估计概率分布。
- 分支级聚合：从通道级熵到分支级熵，需要取平均或加权平均。
- 结构约束：不是所有低熵分支都可以直接删除，必须考虑网络拓扑是否还能正确 forward。

## 7. Step 5：构造信息熵裁剪模型

### 这一步做了什么

根据信息熵排序结果修改 YOLOv8n 的 YAML 结构，将 Concat 11 的输入从双输入改为单输入：

```yaml
Before: [[-1, 6], 1, Concat, [1]]
After:  [[-1],    1, Concat, [1]]
```

对应含义是删除：

```text
Layer 6 -> Concat 11
```

最终生成的信息熵裁剪结构文件为：

```text
07_experiments/stage1_algorithm_baseline_pruning/A2_entropy_pruning/yolov8n_entropy_pruned.yaml
```

### 这一步的作用

这一步把“分支选择”真正转化为“网络结构改变”。只有修改 YAML 并验证模型可以 build 和 forward，裁剪才算从分析阶段进入可训练模型阶段。

### 论文中的体现

对应 Chapter 3 §3.2 的 Figure 2：FPN Topology Before/After Entropy-Guided Pruning。

Figure 2 应体现：

- Baseline 中 Layer 6 skip branch 与上采样主路径一起进入 Concat 11。
- Pruned 模型中 Layer 6 skip branch 被删除。
- Concat 11 变成单输入传递节点，后续 Layer 12 C2f 的输入通道相应重配置。

### 背后的知识点

- YOLO YAML 中每一层通常表示为 `[from, repeats, module, args]`。
- `from=[-1, 6]` 表示当前层同时接收上一层输出和第 6 层输出。
- 删除 Concat 输入会改变后续层输入通道数，因此必须让模型重新 build，检查通道是否匹配。
- 剪枝不仅是删边，还会影响接收层卷积权重形状。

## 8. Step 6：模型 Build、Forward 与权重迁移

### 这一步做了什么

验证裁剪后的模型结构能否正常构建和前向传播，并从 baseline 权重迁移可复用参数。

当前结果：

```text
Model build: OK
Forward pass: OK
Transferred 354/355 items from pretrained weights
```

裁剪后模型复杂度：

| Model | Parameters | GFLOPs |
|---|---:|---:|
| Entropy-pruned YOLOv8n | 2,995,634 | 8.1 |

### 这一步的作用

这一阶段检查两个问题：

1. 结构是否正确：能否 build 和 forward。
2. 初始化是否合理：大部分 baseline 权重能否迁移到裁剪后模型中。

如果直接随机初始化裁剪模型，fine-tune 结果会不稳定，也不利于与 baseline 进行公平比较。

### 论文中的体现

论文中不需要展开写“Transferred 354/355”，但可以在实验设置中说明：

```text
The pruned model was initialized by transferring compatible weights from the trained YOLOv8n baseline and was subsequently fine-tuned under the same data configuration.
```

### 背后的知识点

- 结构剪枝后，部分层的权重形状会改变，不能完全加载旧 checkpoint。
- 可匹配权重迁移能够保留 baseline 已学习到的特征表达。
- Fine-tune 的作用是让结构改变后的模型重新适应数据分布。

## 9. Step 7：Entropy 裁剪模型 Fine-tune 与最终验证

### 这一步做了什么

对信息熵裁剪后的模型进行 50 epoch fine-tune，并进行最终独立验证。

当前 Table 2 固化结果：

| Strategy | Branch Pruned | mAP@0.5 | mAP@0.5:0.95 | Parameters | GFLOPs |
|---|---|---:|---:|---:|---:|
| Entropy-Guided Pruning | L6 to C11 | 0.765 | 0.440 | 2,995,634 | 8.1 |

### 这一步的作用

预验证阶段的模型精度较低，说明“结构能跑”不等于“模型能用”。Fine-tune 是结构裁剪后恢复精度的必要步骤。

最终结果表明，在当前设置下，信息熵裁剪模型的 mAP@0.5 保持在 baseline 附近，甚至略高于 baseline；但 mAP@0.5:0.95 从 0.447 降到 0.440，说明更严格定位指标略有损失。

### 论文中的体现

对应 Chapter 4 §4.2 和 Table 2。

应谨慎写成：

```text
After 50 epochs of fine-tuning, the entropy-pruned model achieved 0.765 mAP@0.5 and 0.440 mAP@0.5:0.95. These results indicate that the selected FPN branch can be removed while maintaining detection performance close to the baseline under the evaluated setting.
```

不建议写成：

```text
Entropy pruning significantly outperforms random pruning.
```

因为当前随机裁剪均值 mAP@0.5 为 0.7653，与 Entropy 的 0.765 几乎相同。

### 背后的知识点

- Fine-tuning：在已有权重基础上用较短训练恢复模型性能。
- mAP@0.5 与 mAP@0.5:0.95 可能呈现不同趋势。
- 单次实验不能证明统计显著性，尤其当差值小于随机实验标准差时。

## 10. Step 8：Random Pruning 三次对照实验

### 这一步做了什么

随机选择三个不同分支进行裁剪，并分别 fine-tune 和验证，用于判断 Entropy 选择是否只是随机运气。

当前三次随机裁剪结果：

| Run | Branch Removed | mAP@0.5 | mAP@0.5:0.95 | Parameters |
|---|---|---:|---:|---:|
| Run 1 | L9 to C20 | 0.767 | 0.442 | 2,946,482 |
| Run 2 | L4 to C14 | 0.763 | 0.438 | 3,007,922 |
| Run 3 | L12 to C17 | 0.766 | 0.443 | 2,995,634 |

随机裁剪均值：

| Metric | Mean | Sample Std |
|---|---:|---:|
| mAP@0.5 | 0.765333 | 0.002082 |
| mAP@0.5:0.95 | 0.441000 | 0.002646 |

### 这一步的作用

Random pruning 是最重要的“反证对照”之一。它回答的问题是：

```text
如果随便删一个分支，结果会不会也差不多？
```

当前结果说明：在当前实验设置下，随机裁剪也能维持 baseline 附近的 mAP@0.5，而且均值与 Entropy 非常接近。因此论文必须承认这个结果边界。

### 论文中的体现

对应 Chapter 4 §4.2 的 Table 2。

建议写法：

```text
The random-pruning baseline obtained an average mAP@0.5 of 0.7653 over three runs. The entropy-guided result is comparable to this average rather than clearly superior, suggesting that the current pruning budget is mild and that a larger search space or hardware-cost-aware criterion may be needed to further separate pruning strategies.
```

这句话很重要，因为它让论文更诚实，也更像真正的科研，而不是硬拗结果。

### 背后的知识点

- 随机对照用于排除“随便做也有效”的可能。
- 多次随机实验要报告 mean 和 std。
- 当方法差异小于或接近随机标准差时，不能宣称显著优势。

## 11. Step 9：L1-Norm Pruning 对照实验

### 这一步做了什么

根据接收层 C2f 中与 skip branch 对应的卷积权重 L1 范数进行分支排序，选择 L1 score 最低的分支裁剪。

当前 L1 选择结果：

```text
Selected branch: Layer 4 -> Concat 14
```

该结构与 Random Run 2 完全相同，因此复用 Random Run 2 的训练和验证结果：

| Strategy | Branch Pruned | mAP@0.5 | mAP@0.5:0.95 | Parameters |
|---|---|---:|---:|---:|
| L1-Norm Pruning | L4 to C14 | 0.763 | 0.438 | 3,007,922 |

### 这一步的作用

L1-Norm pruning 是传统权重幅值剪枝基线，用于与信息熵这种“激活特征分布”指标进行对照。

L1 关注的是权重大小，而 Entropy 关注的是前向传播中特征图的空间信息分布。两者衡量对象不同：

```text
L1-Norm: 参数权重是否小
Entropy: 激活特征是否信息丰富
```

### 论文中的体现

对应 Chapter 4 §4.2 的 Table 2。

建议说明：


## 17. 阶段一背后的核心知识点地图

### 17.1 YOLOv8n 与 Neck 结构

- Backbone：提取不同尺度的基础视觉特征。
- Neck：融合多尺度特征，增强检测不同尺寸目标的能力。
- Head：输出分类和边界框预测。
- FPN/PAN：通过上采样、下采样和跨层连接融合语义与空间信息。

### 17.2 FPN 分支为什么会影响硬件

FPN skip branch 在算法上有助于多尺度融合，但在 FPGA 上意味着：

- 早期特征图需要被保存到后面使用。
- 大尺寸 feature map 会增加 BRAM 或 DDR 压力。
- Concat 会增加后续卷积输入通道数。
- 跨层搬运会增加数据流调度复杂度。

所以阶段一的裁剪不是单纯为了减少参数，而是为了减少低收益、高代价的数据流。

### 17.3 信息熵为什么可以作为分支指标

信息熵衡量特征分布的不确定性。对于一个空间特征图，如果其激活分布变化较丰富，直方图概率分布更分散，熵可能更高；如果激活分布更集中，熵可能更低。

但信息熵不是万能指标：

- 熵高不一定代表对检测任务更重要。
- 熵低也不一定代表可以删除。
- 最终必须通过 fine-tune 后的 mAP 验证。

### 17.4 Random 和 L1 对照为什么必要

Random pruning 回答：

```text
你的方法是不是只是刚好删了一个也没影响的分支？
```

L1 pruning 回答：

```text
传统权重幅值指标是否已经足够？
```

如果没有这两个对照，Entropy pruning 的结论会很弱。

### 17.5 Fine-tune 为什么必要

结构裁剪改变了特征融合路径。即使大部分权重可以迁移，模型仍然需要重新适应新的特征流，因此 fine-tune 是裁剪实验中恢复精度的关键步骤。

### 17.6 Golden 数据为什么重要

HLS 阶段无法直接“相信”硬件输出正确。必须有一个来自 PyTorch 的逐元素参考输出作为 golden output。

后续阶段二的正确性判断应写成：

```text
HLS output vs golden_output.txt MSE ≤ 1e-6
```

而不是：

```text
HLS output vs floating-point PyTorch output MSE ≤ 1e-6
```

因为当前 golden output 已经是 Q8.8 定点化后的参考。
