## 阶段一：YOLO Baseline 建立与信息熵裁剪

### 目标关键字

  ```

### 知识脉络

本阶段相当于算法学习中的“基础模板题”。

你要从“能跑 YOLO”推进到“能解释 YOLO/FPN 的结构瓶颈”。

核心理解链条：

```text
YOLO 需要检测大、中、小目标
        ↓
所以 Neck 部分使用 FPN/PAN 做多尺度特征融合
        ↓
FPN/PAN 中存在上采样、Concat、跳连、3×3 Conv
        ↓
这些结构提升检测精度，但会产生大量中间 Feature Map 搬运
        ↓
长跳连需要保存早期 Feature Map，增加 BRAM / DDR 压力
        ↓
不是所有跳连分支都同等重要
        ↓
用信息熵衡量不同分支的特征信息量
        ↓
裁剪低信息量且硬件代价高的分支
```

你要建立的学术逻辑是：

```text
裁剪不是为了单纯减少参数
而是为了减少低信息收益、高硬件代价的 FPN 数据流
```

### 必须完成

-  下载并整理数据集，例如 NEU-DET 或遥感小目标数据集
-  跑通 YOLOv5/v8 baseline 训练
-  记录原始模型指标：
  - `mAP@0.5`
  - 参数量
  - FLOPs
  - 推理时间
-  定位 YOLO Neck / FPN / PAN 中的关键融合层
-  提取多尺度融合层 Feature Map
-  编写信息熵计算脚本
-  对每条跳连分支计算平均熵：

```text
X^(k) ∈ R^(C_k × H_k × W_k)

H_c^(k) = -Σ p_c(b) log2(p_c(b) + ε)

H_bar^(k) = (1 / C_k) Σ H_c^(k)
```

-  按信息熵排序，裁剪低信息量分支
-  修改 `yolo.yaml`
-  裁剪后 Fine-tune 30-50 epoch
-  实现三组裁剪对照：
  - Random Pruning
  - L1-Norm Pruning
  - Entropy Pruning
-  输出 Table 2：

```text
Pruning Strategy | Branches Pruned | mAP@0.5 | ΔmAP | FLOPs | Params
```

-  导出 HLS 所需数据：
  - `golden_input.txt`
  - `golden_output.txt`
  - `weights.txt`

对应章节：

### 知识脉络

本阶段相当于算法学习中的“线段树、图论、动态规划难题”。

你不能只会概念，必须把数据流写成硬件友好的 C++ 结构。

核心理解链条：

```text
输入 Feature Map 从 DDR 读入
        ↓
不能每个 3×3 窗口都重新读 DDR
        ↓
Line Buffer 缓存 K-1 行
        ↓
Window Buffer 实时形成 3×3 滑动窗口
        ↓
窗口像素和权重进入 PE/MAC 阵列
        ↓
Adder Tree 累加多个乘法结果
        ↓
BN/ReLU 后处理
        ↓
输出 Feature Map
```

进一步的硬件逻辑：

```text
Line Buffer 解决行复用
Window Buffer 解决窗口复用
PE/MAC 解决乘加并行
PIPELINE 解决吞吐
ARRAY_PARTITION 解决数组端口冲突
AXI4-Stream 解决流式输入输出
ap_fixed 解决浮点硬件代价过高
```

本阶段必须把软件卷积思维转成硬件流水线思维：

```text
软件思维：for 循环逐点计算
硬件思维：每个 clock 推进一批数据，多个阶段并行工作
```

### 必须完成

-  用 C++ 重写核心融合算子：
  - Concat
  - 3×3 Conv
-  将数据类型从 float 改成：

```cpp
ap_fixed<16, 8>
```

-  实现 AXI4-Stream 输入输出接口
-  独立验证 Line Buffer：
  - 缓存 `K-1` 行
  - 验证行滚动逻辑
-  独立验证 Window Buffer：
  - 形成 3×3 窗口
  - 验证窗口移位逻辑
-  实现 PE/MAC 阵列
-  实现 Adder Tree
-  可选实现 BN/ReLU
-  加入 HLS pragma：

```cpp
#pragma HLS PIPELINE
#pragma HLS ARRAY_PARTITION
#pragma HLS INTERFACE axis
#pragma HLS DATAFLOW
```

-  C Simulation 通过
-  HLS 输出和 `golden_output.txt` 对齐
-  C Synthesis 通过
-  记录资源报告：
  - LUT
  - FF
  - DSP

  - II
-  尽量完成 C/RTL Cosimulation
