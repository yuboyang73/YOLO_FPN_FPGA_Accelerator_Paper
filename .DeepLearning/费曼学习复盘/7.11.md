## 1. 神经网络为什么不能只堆叠线性变换？

### 问题

如果每一层都只有：

```text
z = Wx + b
```

没有 ReLU / Sigmoid 这类激活函数，多层网络最后为什么仍然等价于一个线性模型？

### 我的回答

因为如果只有线性模型，那么无论多少层都只能表达线性的意思，无法反映更多复杂的特征。

### 判断

正确。

### 标准理解

多层线性变换叠加后仍然是线性变换。例如：

```text
y = W2(W1x + b1) + b2
```

展开后：

```text
y = (W2W1)x + (W2b1 + b2)
```

仍然可以写成：

```text
y = W'x + b'
```

所以，如果没有激活函数，多层网络本质上仍然等价于一层线性模型。激活函数的作用是引入非线性表达能力，让网络可以拟合更复杂的特征关系。

### 出错点

本题没有明显错误。

## 2. Softmax + 交叉熵到底在惩罚什么？

### 问题

假设真实类别是第 3 类，模型给第 3 类的概率从 `0.8` 降到 `0.1`，交叉熵损失会怎样变化？请用“真实类别概率”解释。

### 我的回答

一开始不太理解。经过讲解后回答：

交叉熵越大，真实类别的概率就很低；交叉熵越小，真实类别的概率高。

### 判断

基本正确。

### 标准理解

Softmax 先把模型输出的类别分数转换为概率。交叉熵用于惩罚模型没有把足够高的概率分给真实类别。

当真实类别为第 `c` 类时，交叉熵可以写成：

```text
L = -log(y_c)
```

其中 `y_c` 是真实类别的预测概率。

如果：

```text
y_c = 0.8
L = -log(0.8) ≈ 0.223
```

如果：

```text
y_c = 0.1
L = -log(0.1) ≈ 2.303
```

所以，真实类别概率越低，交叉熵损失越大；真实类别概率越高，交叉熵损失越小。

### 出错点

一开始对 Softmax 和交叉熵的分工不清楚：

- Softmax：把类别分数转换为概率。
- 交叉熵：惩罚真实类别预测概率过低。

## 3. 梯度下降中的梯度方向表示什么？

### 问题

如果某个参数 `w` 的梯度是正数，为什么更新时要让 `w` 变小？请把“梯度正负”和“损失变化方向”说清楚。

### 我的回答

梯度为正时，`w` 增大，`L` 增大，我们的目标是让损失 `L` 减小，因此要让 `w` 减小。

反过来，`w` 增大，`L` 会减少。根据公式：

```text
w <- w - η * (负数)
```

我说成了：`w` 会加上一个负数，因此 `w` 会变大。

### 判断

前半部分正确，后半部分有公式符号错误。

### 标准理解

梯度表示损失函数对参数变化的敏感程度：

```text
∂L / ∂w
```

如果梯度为正：

```text
∂L / ∂w > 0
```

表示 `w` 增大时，损失 `L` 会增大。为了让损失减小，应该让 `w` 减小。

如果梯度为负：

```text
∂L / ∂w < 0
```

表示 `w` 增大时，损失 `L` 会减小。因此更新时应该让 `w` 增大。

梯度下降公式：

```text
w <- w - η * ∂L/∂w
```

当梯度为负时：

```text
w <- w - η * (负数)
```

等价于：

```text
w <- w + 一个正数
```

所以 `w` 会变大。

### 出错点

把“减去负数”说成了“加上负数”。

正确说法是：

```text
减去负数 = 加上正数
```

## 4. 反向传播为什么需要链式法则？

### 问题

为什么网络越深，不能只看最后一层的误差，而要把误差一层一层传回前面的参数？

### 我的回答

最后的损失确实反映了预测错误，但是不能告诉我每一层、每一个参数应该怎么改，因此必须要经过链式传播向前传递，改变前面每一层的参数。

### 判断

核心正确，但表达里“向前传递”应改为“反向传播”。

### 标准理解

最终损失只能说明模型错了多少，但不能直接说明每一层参数应该如何修改。

前面层的参数通过下面这条路径间接影响损失：

```text
前面参数 -> 中间特征 -> 最终预测 -> 损失
```

所以要计算前面参数对损失的影响，必须通过链式法则把每一层的局部影响连起来：

```text
损失对输出的影响
× 输出对中间特征的影响
× 中间特征对前面参数的影响
```

反向传播就是从损失函数开始，沿着网络反方向逐层计算梯度。

### 出错点

把反向传播说成了“向前传递”。

正确边界：

- 前向传播：输入到输出。
- 反向传播：损失到前面层参数。

## 5. ReLU 的反向传播有什么风险？

### 问题

当输入 `x <= 0` 时，ReLU 梯度为 0，这会对某些神经元的学习造成什么影响？Leaky ReLU 又是怎么缓解这个问题的？

### 我的回答

ReLU 只当 `x > 0` 的时候才有数，如果当 `x < 0` 时候会将其所有的参数都丢掉，可能会丢掉一些有用的参数。

### 判断

错误。

### 标准理解

ReLU 的前向传播为：

```text
ReLU(x) = max(0, x)
```

反向传播中：

```text
x > 0  时，梯度为 1
x <= 0 时，梯度为 0
```

ReLU 不会删除参数。它的问题是：当某个神经元长期处在负半区时，它的输出一直是 0，反向梯度也一直是 0，相关参数就很难继续更新。这叫 dying ReLU 问题。

Leaky ReLU 在负半区保留一个很小的斜率：

```text
LeakyReLU(x) = αx, x < 0
```

这样即使 `x < 0`，仍然有小梯度可以传回去，神经元不容易彻底失活。

### 出错点

把“梯度阻断”误解成了“参数被丢掉”。

正确说法：

```text
ReLU 在负半区不会删除参数，而是让该路径当前的梯度为 0。
```

## 6. 卷积层和全连接层相比，为什么更适合图像？

### 问题

请从局部连接、权值共享、参数量、空间结构中至少选 3 个，解释卷积为什么适合提取图像特征。

### 我的回答

第一次回答：

卷积层可以通过不同的卷积核提取不同的特征，在局部连接方面会更有优势，能够将图像的不同特征分析清楚。

修正后回答：

局部连接：图像相邻像素关系强，卷积核只看局部区域，符合图像的空间结构。

权值共享：同一个卷积核在整张图上滑动，能在不同位置检测同一种特征。

### 判断

修正后正确。

### 标准理解

卷积比全连接更适合图像，主要因为：

1. 局部连接：图像相邻像素之间相关性强，卷积核只关注局部区域，符合图像空间结构。
2. 权值共享：同一个卷积核在整张图上滑动，可以在不同位置检测同一种特征。
3. 参数量更少：卷积核尺寸远小于整张图像，不需要像全连接层那样连接所有像素。
4. 保留空间结构：卷积输出仍然是特征图，保留了高、宽、通道等结构信息。

### 出错点

第一次回答只说了“卷积核提取不同特征”，但没有明确讲出：

- 局部连接为什么适合图像。
- 权值共享为什么能在不同位置检测同类特征。
- 参数量为什么比全连接更少。

## 7. 卷积 + BN 为什么能在推理阶段融合？

### 问题

BN 在推理阶段为什么可以看成固定线性变换 `y = az + c`？融合成 `W' = aW, b' = ab + c` 后，对 FPGA 推理加速有什么好处？

### 我的情况

这题没有直接作答，要求进行讲解；随后追问了 BN 的作用。

### 标准理解：BN 的作用

BN，即 Batch Normalization，核心作用是让中间特征分布更稳定。

它先标准化：

```text
z_norm = (z - μ) / sqrt(σ² + ε)
```

再缩放和平移：

```text
y = γ * z_norm + β
```

其中：

- `μ`：均值。
- `σ²`：方差。
- `ε`：防止除零的小常数。
- `γ`：可学习缩放参数。
- `β`：可学习平移参数。

BN 的主要作用：

1. 稳定每层中间特征分布。
2. 让训练更稳定。
3. 有时可以允许使用较大学习率。
4. 有一定正则化效果。
5. 推理阶段可以和卷积融合。

### 标准理解：为什么推理阶段可以融合

BN 推理公式：

```text
BN(z) = γ * (z - μ) / sqrt(σ² + ε) + β
```

推理阶段，`γ`、`β`、`μ`、`σ²` 都是训练后固定下来的值，因此可以整理成：

```text
BN(z) = a*z + c
```

其中：

```text
a = γ / sqrt(σ² + ε)
c = β - γμ / sqrt(σ² + ε)
```

如果卷积输出为：

```text
z = Wx + b
```

那么：

```text
y = BN(z)
y = a(Wx + b) + c
y = (aW)x + (ab + c)
```

所以可以定义新的卷积参数：

```text
W' = aW
b' = ab + c
```

原来的：

```text
Conv -> BN
```

可以融合成：

```text
Conv'
```

### FPGA 推理加速意义

融合后不需要单独实现 BN 模块，可以减少：

- 额外乘法和加法。
- 中间特征图读写。
- 独立 BN 层控制逻辑。
- 流水线中的额外阶段。

这对 FPGA 推理加速有帮助，因为它减少了计算和数据搬运，让硬件数据流更连续。

### 出错点

本题没有直接回答，所以主要遗漏是：

- 没有说明 BN 在推理阶段使用固定统计量。
- 没有说明 BN 可以化成固定线性变换。
- 没有说明 Conv + BN 融合可以减少独立 BN 计算和中间数据搬运。

## 8. 训练和硬件推理加速的边界在哪里？

### 问题

结合论文方向回答：深度学习训练负责得到什么？FPGA/HLS 推理加速主要优化什么？为什么 BN 融合、剪枝、减少特征图数据搬运主要属于推理阶段优化？

### 我的情况

本题还没有正式回答。

### 标准理解

训练阶段的目标是通过数据和损失函数学习模型参数，例如：

- 卷积权重。
- 偏置。
- BN 的 `γ` 和 `β`。
- 分类头或检测头参数。

推理阶段的目标是使用已经训练好的参数完成预测。FPGA/HLS 推理加速主要优化的是：

- 计算速度。
- 延迟。
- 吞吐率。
- 片上缓存使用。
- 片外访存次数。
- 数据搬运带宽。
- DSP、BRAM、LUT 等硬件资源消耗。

BN 融合、剪枝、减少特征图数据搬运主要属于推理阶段优化，因为它们通常不负责重新学习模型，而是让已经训练好的模型以更低成本运行。

在你的论文方向中，可以这样理解：

```text
训练阶段：得到可用的 YOLO/FPN 检测模型参数
推理加速阶段：让这个模型在 FPGA/HLS 上更快、更省资源地运行
```

熵剪枝和 FPN 特征图优化的核心意义是减少不必要的特征通道或数据流，从而降低硬件部署时的访存压力和计算压力。

### 出错点

本题尚未作答，后续复习时需要重点训练。

## 本轮总体易错点

1. Softmax 和交叉熵的分工容易混淆。
2. 梯度为负时，公式是“减去负数”，也就是“加上正数”。
3. 反向传播方向是从损失到前面层，不是向前传播。
4. ReLU 负半区不是删除参数，而是阻断梯度。
5. BN 推理融合依赖固定均值和方差，训练阶段不能简单提前融合。
6. 硬件推理加速优化的是部署运行成本，不是重新训练模型本身。
